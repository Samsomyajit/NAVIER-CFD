# 智能体 AI

NAVIER-CFD 的智能体层用于将自然语言研究目标转换为可检查、可复现的 CFD 实验计划。

## 主要职责

智能体可以协助：

- 明确问题、变量、几何和物理范围；
- 推荐候选数据集和模型；
- 生成训练与基准配置；
- 识别数据泄漏、指标不足和资源约束；
- 输出实验清单、消融计划和报告结构；
- 连接模型推荐结果与执行工作流。

## 离线确定性规划

```python
from navier_cfd import AgentOrchestrator

plan = AgentOrchestrator().plan_json(
    "在 24 GB GPU 上比较 RealPDEBench 圆柱预测，要求守恒和不确定性评估"
)
```

命令行：

```bash
navier agent plan \
  "Benchmark RealPDEBench cylinder forecasting with 24 GB VRAM"
```

## 设计原则

### 1. 智能体不能替代数值验证

语言模型提出的架构、损失或超参数必须通过代码测试、网格无关性检查、物理残差和正式测试集验证。

### 2. 所有建议必须可审计

实验计划应记录：

- 目标与假设；
- 数据集和修订版本；
- 模型及构建参数；
- 训练预算；
- 数据划分；
- 指标；
- 随机种子；
- 停止条件；
- 风险和失败判据。

### 3. 区分代理、校正器和求解器

智能体不得把全场代理模型的速度提升与嵌入求解器的校正器或预条件器直接比较。

### 4. 保留人在回路中

数据许可、论文复现声明、高成本训练、外部代码安装和结果发布需要研究人员确认。

## 推荐使用方式

1. 用 `TaskSpec` 明确任务；
2. 运行证据感知推荐器；
3. 由智能体生成候选实验计划；
4. 用数据集适配器和 `Experiment` 执行；
5. 保存检查点、运行清单和指标；
6. 由研究人员审查并解释结果。
