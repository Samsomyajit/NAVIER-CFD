# 混合加速案例

本案例研究神经网络如何作为校正器、预条件器或求解器内模块加速传统 CFD，而不是完全替代数值求解器。

## 代表方法

- Solver-in-the-Loop；
- Indirect Neural Corrector（INC）；
- NeuroSEM；
- 神经算子预条件 Newton；
- 几何感知神经预条件器；
- PICT、diffSPH 等专业外部运行时。

## 评价协议

必须同时报告：

- 数值误差与守恒；
- 收敛迭代数；
- 稳定性和失败率；
- 单步运行时间；
- 完整求解时间；
- 训练成本；
- 多次部署后的盈亏平衡点。

## 公平比较

神经校正器应与相同网格、相同收敛容差和相同硬件上的传统求解器比较。不能只比较网络推理时间而忽略数据传输、预处理、求解器步骤和训练成本。

## 检查点

建议保留传统求解器基线、混合求解器、纯代理模型三组结果，并报告超出训练工况时的稳定性。
